## 三、Agent Harness：被低估的中间件战争

### 发生了什么

**智能体 Harness 成为关键竞争层**：
> "用户随口说'LLM'时，实际使用的是集成**格式化、解析器、工具调用、结构化生成、记忆**的智能体语言系统。" —— voooooogel

**核心组件**：
1. **编排层**：任务分解、依赖管理、并行调度（如Cline Kanban）
2. **记忆层**：跨会话持久记忆、共享记忆池（Hermes Agent vs OpenClaw）
3. **工具接口**：MCP协议成为事实标准，Figma已原生支持
4. **安全策略**：沙箱、权限控制、审计日志
5. **评估循环**：追踪、测试、事故反馈

**LangSmith闭环**：
> "close the loop with traces, evals, incidents, and production feedback"

### 为什么Harness比模型更重要

**实践痛点**：
- GPT-5.2 Pro子智能体过于激进，常委托给更弱的子智能体
- Claude Computer Use仍显脆弱，速度和稳定性待提升
- 并行化常沦为"slop theater"（浮夸演示）而非真正的吞吐量提升

**真实瓶颈**：
| 层级 | 问题 | 解决方案方向 |
|------|------|--------------|
| 基础设施 | 推理延迟 | 预取、缓存、边缘部署 |
| 编排 | 任务冲突 | 依赖图、锁机制、合并策略 |
| 工具 | API不可靠 | 重试、回退、模拟执行 |
| 评估 | 难以验证 | 在线A/B测试、影子模式 |

### Figma MCP：工具原生化的里程碑

**2026年3月发布**：
- Figma MCP服务器进入公开测试
- GitHub Copilot CLI可直接通过MCP编辑Figma画布
- Cursor立即扩展支持基于团队设计系统生成前端组件

**意义**：工具调用从"聊天包装器原生"走向"产品原生"

**AI在银行的价值创造**：
- 全球银行业年度潜在价值：**2000-3400亿美元**

- 误报率降低：**60%**

### Agentic AI的2026分水岭

**从"推荐"到"执行"**：
- **交易执行**：AI智能体可发起交易、再平衡投资组合
- **贷款处理**：从申请到放款的全自动工作流
