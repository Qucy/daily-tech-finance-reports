# 📊 科技金融日报 - 时间过滤与去重指南

## ⚠️ 问题诊断

**发现的问题：**
- 搜索结果包含大量过期新闻（2025年3月、甚至更早）
- 没有严格验证新闻发布日期与当前时间的差异
- 导致日报中出现大量过期内容

## ✅ 正确的操作流程

### 步骤1：获取当前时间（必须！）
每次生成报告前，**首先**调用：
```
session_status
```
获取当前系统时间（例如：2026-03-25）

### 步骤2：计算时间窗口
- **当前日期**：2026-03-25（从session_status获取）
- **3天前日期**：2026-03-22（当前日期 - 3天）
- **有效新闻范围**：发布日期 >= 2026-03-22

### 步骤3：搜索并过滤新闻

#### 对于每条搜索结果：
1. **提取发布日期**（从标题、URL、或内容中）
2. **验证日期格式**：
   - ✅ 2026-03-25（有效）
   - ✅ 2026-03-24（有效）
   - ✅ 2026-03-23（有效）
   - ✅ 2026-03-22（有效）
   - ❌ 2026-03-21（过期，丢弃！）
   - ❌ 2025-03-20（过期，丢弃！）
   - ❌ 2024-XX-XX（过期，丢弃！）

3. **只保留3天内的新闻**

### 步骤4：去重检查（在有效新闻中）
对于保留的新闻，检查过去7天是否已报道：
- 如果是重复内容 → 只保留标题 + 标注历史日期
- 如果是新内容 → 完整描述

### 步骤5：生成报告
汇总过滤后的新闻，生成日报

## 🔍 日期提取规则

### 从URL提取日期
| URL模式 | 提取的日期 |
|---------|-----------|
| `.../2026-03-25/...` | 2026-03-25 |
| `.../20260325/...` | 2026-03-25 |
| `.../mar-25-2026/...` | 2026-03-25 |

### 从标题提取日期
| 标题 | 提取的日期 |
|------|-----------|
| "XXX发布（2026-03-25）" | 2026-03-25 |
| "2026年3月25日：XXX" | 2026-03-25 |
| "XXX - March 25, 2026" | 2026-03-25 |

### 从内容提取日期
查找内容中的日期字符串，优先选择：
- 发布日期（Published/Released）
- 创建日期（Created）
- 最后更新日期（Updated）

## ❌ 常见错误示例

### 错误1：没有获取当前时间
```
❌ 直接使用搜索结果的日期，不验证是否过期
```

### 错误2：接受所有搜索结果
```
❌ 将所有搜索结果都加入日报，不做时间过滤
```

### 错误3：错误的年份判断
```
❌ 将2025年的新闻误认为是最近的新闻
```

## 🔧 技术实现要点

### 伪代码
```python
# 1. 获取当前时间
current_date = get_session_status().time  # 2026-03-25
three_days_ago = current_date - 3_days    # 2026-03-22

# 2. 搜索新闻
search_results = search("AI news")

# 3. 过滤新闻
valid_news = []
for news in search_results:
    news_date = extract_date(news)  # 从标题/URL/内容提取日期
    
    if news_date is None:
        continue  # 丢弃无日期的新闻
    
    if news_date < three_days_ago:
        continue  # 丢弃过期新闻
    
    # 检查是否重复
    dup_date = check_duplicate_in_past_7_days(news.title)
    if dup_date:
        news.is_duplicate = True
        news.dup_date = dup_date
    
    valid_news.append(news)

# 4. 生成报告
generate_report(valid_news)
```

1. **必须调用 session_status** 获取当前时间
2. **必须计算3天前日期** 作为过滤阈值
3. **必须验证每条新闻的日期** 在阈值之后
4. **必须丢弃无法验证日期的新闻**
5. **必须在有效新闻中进行7天去重**
